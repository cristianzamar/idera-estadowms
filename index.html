Recomendaciones para normalizar los servidores WMS

Acerca de los atributos WMS del Servidor

Atributo Title

Se recomienda completar este atributo a fin de que este texto sea descriptivo para quien esté viendo el listado de servicios WMS en el visualizador de IDERA.

Atributo Abstract

Se recomienda completar este atributo a fin de que describa los datos provistos por el servicios WMS.

Acerca del puerto 80 del Servidor

Es recomendable que el servicio WMS esté accesible a través del puerto 80 (el puerto estándar para la web) a fin de que los usuarios que deseen consultar los servicios WMS no se vean imposibilitados por algún proxy/firewall. Son muy comunes los proxies institucionales que solo permiten a los usuarios de la red local, el acceso a sitios o páginas en el puerto estándar de la web.

Acerca de los atributos WMS de las capas

Atributo Title

Se recomienda completar este atributo a fin de que este texto sea descriptivo de los datos que la capa contiene. En general, el software de servidor suele completar automáticamente este atributo con el mismo valor del atributo name. Se recomienda que estos dos valores difieran ya que el atributo nombre se utiliza como identificador único y el título como descripción del identificador.

Atributo Abstract

Se recomienda completar este atributo a fin de que describa los datos y tipos de datos contenidos en esta capa.

Atributo Keywords (Palabras clave)

Este campo no solo permite palabras individuales. Sino que cada keyword puede ser un término compuesto como por ejemplo Aguas interiores. Se recomienda aprovechar este atributo para que contenga entre las keywords, por lo menos uno de los temas definidos por el perfil de metadatos de IDERA. Estos temas son Agricultura, Biota, Límites, Clima, Economía, Elevación, Medio ambiente, Información geocientífica, Salud, Coberturas Básicas, Inteligencia / Militar, Aguas interiores, Ubicación, Océanos, Planificación de catastro, Sociedad, Estructura, Transporte, Utilidades / Comunicaciones.
Se recomienda el comienzo de cada tema con la primer letra en mayúscula tal cual las define el perfil de metadatos.

Atributo Link a metadatos

Se recomienda que este campo contenga el link a un servicio de metadatos conforme a OGC que contenga los metadatos para el set de datos contenidos en esta capa.

Acerca de los Sistemas de Referencia Espacial

EPSG:4326

Este código es el código utilizado y reconocido por EPSG para representar datos geoespaciales basado en un dátum (elipsoide y marco de referencia) WGS84 (o ITRF05). En este sistema, la proyección corresponde a una en la cual las coordenadas de un objeto en una imagen en sus ejes X e Y se corresponden con los valores de las coordenadas longitud y latitud de los objetos geográficos, respectivamente.

EPSG:3857

Este código es el código utilizado y reconocido por EPSG para representar el sistema de referencia espacial utilizado por Google Maps, Bing Maps, Yahoo Maps, OpenStreetMap, ESRI Web Maps y demás mapas web populares. Consiste en una proyección de Mercator y se vale de una esfera auxiliiar para la proyección aunque está basado en un datum igual al definido por el WGS84 (o ITRF05).

Soportar este Sistema de Referencia espacial ofrece alta compatilibidad con servicios de mapas a nivel internacional debido a su uso tan común

EPSG:900913

Este código es el que una gran cantidad de software geoespacial (Openlayers, Mapserver, Geoserver, etc) utilizaban para representar el sistema de referencia espacial de Google Maps, Bing Maps, Yahoo Maps, OpenStreetMap, ESRI Web Maps y demás mapas web populares hasta que EPSG la reconoció entre sus datos y le asignó el código EPSG:3857 para definirla.

Soportar este Sistema de Referencia espacial ofrece alta compatilibidad con servicios de mapas a nivel internacional debido a su uso tan común

Gauss-Krüger / POSGAR 94

Esta denominación hace referencia a la proyección Gauss Krüger utilizando el marco de referencia POSGAR 94, que corresponde a los códigos EPSG: 22181, 22182, 22183, 22184, 22185, 22186 y 22187, respectivamente, para las fajas 1 a 7 a fin de representar el territorio nacional en sus correctas proporciones.

Acerca de los Formatos soportados para GetFeatureInfo

text/html

Este formato es el aprovechado por el visualizador de IDERA. Y permite representar la consulta puntual acerca de un objeto geográfico sobre el globito que se despliega al acerca click sobre el mapa.

GML 2

Este formato es un derivado de XML que soporta campos específicamente geoespaciales. Se utiliza a fines de consultas programáticas por parte de un software cuya tarea no es necesariamente representar el resultado para ser visto de manera gráfica aunque sí procesable por otrao sistema.

Acerca de los Formatos soportados para las imágenes GetMap

image/jpeg

El formato JPEG de imágenes es el formato más comun utilizado para las capas de los servidores WMS> Este formato no permite representar transparencias y por su alta compresión con pérdidas es generalmente utilizado para datos que no tengan origen vectorial. JPEG utiliza 24 bits para representar el color de cada pixel y define compresión con pérdidas regulable a la hora de guardar la información de la imagen.

image/png

El formato PNG de imágenes permite representar transparencias y es el comunmente utilizado para las capas superpuestas sobre una capa base. Este formato utiliza 24 bits para representar el color de cada pixel y define compresión sin pérdidas a la hora de guardar la información de la imagen.

image/png8

Esta variación del formato PNG permite archivos PNG de menor tamaño (en términos de bytes) ya que en lugar de usar 24 bits para representar el color de cada pixel, utiliza solo 8 para representar el color de cada pixel pero según una paleta de colores no estándar que está guardada en el mismo archivo de imagen. Esta paleta es generada por el servidor al momento de generar la imagen y solo hace referencia a los colores que sí están siendo usados en la imagen y no a todos los colores posibles del mundo real. Esto deviene en imágenes iguales a las logradas con PNG en la mayoría de los casos pero con un tamaño muy inferior al logrado por el formato PNG de 24 bits.
